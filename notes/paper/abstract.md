# Abstract

We study a bounded same-family latent delegation setting in which a frozen large language model keeps ownership of the master residual stream while replacing part of its middle computation with a delegated computation from a frozen smaller model plus learned interface modules. The default pair is Gemma-2 9B as the large model and Gemma-2 2B as the small model, under a single RTX 5090-class GPU budget with frozen backbones throughout.

The project started from a conservative fixed-window substitution and first established a qualified feasibility result: delegated computation clearly improved over skip-only and no-small controls, but not over strong learned bridge baselines in large hidden space. A continuation branch then tested whether the fixed contiguous substitution window was structurally wrong. Real Gemma Phase 1 rejected the legacy `24..29 -> 14..19` split as the best default and produced a stable two-window shortlist: `24..27 -> 14..19` and `24..27 -> 16..18`. A static two-path mixture over those windows was the first pilot result to beat both bridge controls on KL/NLL, and a low-capacity token-wise two-path gate improved further.

The frozen `v0.6.0` token-wise model is the strongest result in the repo. On the original held-out LM-style probe it reaches KL/NLL `0.255739 / 2.980182`, versus `0.288448 / 3.072051` for `bridge_only`; on a fresh untouched holdout it reaches `0.248886 / 3.185004`, versus `0.289564 / 3.295081` for `bridge_only`. Later Idea 5 and Idea 2 branches are informative analysis branches only and do not replace the best-model claim. A bounded generalization suite shows real but mixed external validity: the strongest carryover remains on held-out LM-style evaluation, while multiple-choice task gains are selective rather than broad. The strongest defensible claim is therefore a strong positive result inside the bounded Gemma-2 setting, not a broad downstream-superiority claim.
